LESSON 1:

Basic of Web and HTML

What's coding? Coding is simply a way to communicate with the computers. Computer doesn't have language like human do, and it's stupid. Somehow, we need to interpret our language so that computer can understand us. That's what coding is actually is.

HTML(Hyper Text Markup Language) is the type of file that is commonly used in a webpage. A webpage can also contain plain-text, images, videos, links, but HTML is the brain of a webpage.

How a web actually works

When you surf with browser on your computer, the computer sends data to internet request using the protocol http to the server. The server then finds the appropriate answer and return it again to our computer. Server itself is actually a computer, but it's super strong and it will be a waste of potential if it is used like the regular computer (surfing facebook??).

HTML mark-up languange

HTML consists of many mark-up languages, following the text. The mark-up language doesn't appear on the webpage, but it gives the special effect to the text that it applies to. A mark-up languages has a structure like: <name>text</name>. Some mark-up language need to have open tag and closing tag, but some doesn't. Example of a mark-up language:

1. Bold
The formula for bold is <b>text</b> (b means bold). For example: If we write <b>text</b>, the computer will intepret it as:text

2.Italic
The formula for italic is <em>text</em> (em means emphasis). For example: If we write <em>text</em>, the computer will intepret it as:text

3.Break
Br means break. HTML can't understand enter command. Some text will need to have break to give reader nice space while reading. A long one line text is extremelly exhausting to read. The tag for br is <br>. This is one of tag that doesn't need closing tag. For example: text<br>text will be intepreted as:
text
text

4. Paragraph
Another way to break long line is using <p> symbol (p means paragraph). This one needs closing tag, while break tag doesn't. But there is quite a difference between <br> and <p>. Break just break the line, while paragraph will create some sort of bounding box accross the content of it. This box will have attribute like height and weight. We can classify this into 2: inline tag and block tag. Bold, italic, break are inline tag, while p, div, form are block tag.

5.Links
The formula to create link is <a attr (attribute)>content</a> (a means anchor). For example: If we write <a href="www.udacity.com">text</a>, the word "text" will be a link:text

6.Images
The formula for images is <img attr alt=""> (img means image). For example: If we write <img src="http://goo.gl/Sh9FXJ" alt="html">, computer will display images based on the link we provide: Note that the image will appear inline with the text, and the tag doesn't need closing tag (just like <br>). Alt is the text that will appear when the image couldn't be loaded (maybe wrong links, etc). So alt is not really necessarry, but it is good to have.

Side-note

While knowing these html tag, we already have basic webpage, in order to create an actual website, it needs more than just mark-up languages. The structure of actual webpage is:
<!doctype html>
<html>
<head>
</head>
<body>
<mark-up languages>
</body>
</html>

doctype just the indicator of html page, head will contain title (this will be displayed on the top on the browser tab), javascript, css, meta-datas, fonts, etc. Body is where the mark-up languages is.

LESSON 2:

Creating a Structured Document with HTML

Difference between HTML, CSS, Javascript

HTML file consist of either inline or block. Block element form "invincible box" around the content. The most commonly used is <div>. While the example of inline tag is <span>.
CSS is the tool to style the HTML file, you can adjust padding, color, positions, etc. And Javascript is the tool to modify the existing element to behave like what you want it to be. This three are best friends and work complement each other to create beautiful-stunning website.

Tree-like structure

If you analyze a complete HTML file, you will notice what is called tree-like structure. It means it has branches, and you can dig each branch to go deeper and deeper into the content. A great way to analyze an HTML file is by using Google Chrome > tools > developer tools.

Boxes everywhere

When you analyze HTML file using developer tools from Chrome, you will find that all elements are displayed as boxes, even the non-box such as text, or circle image, they have the "invincible box" around them. Why is that so? Go to the next note.

Boxified work-structure

This boxed-structure leads to a method of boxified work-structure. When a professional web developer sees a page, they will think of boxes: first the main boxes and inside there will be child boxes, and so on untill it becomes the tree-like structure with boxes as the branch. They classify the content with boxes, they sketch or using paper-cuts to imagine the web-page structure. In that way, they can structure the web-page easily before jump in to write code.

Programmer's tools

There are many good tools for writing the program but the most recommended is Sublime text. This software is specially designed for writing codes, it's fast and convenient.
Another great tool is scratchpad.io but it's not suitable if you are writing long codes, it is great for testing out codes because you can directly see the result on your left.
Technically you can use plain-text editor such as textedit or notepad, but they are not meant to be a programmer's tool.
